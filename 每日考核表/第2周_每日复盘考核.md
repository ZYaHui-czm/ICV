## 🗓️ 第8天 <a id="day8"></a>

**主题**：字典 — 最实用的数据结构

**学习内容**：字典创建/CRUD、`get()`vs`[]`、`.items()`遍历、词频统计经典模式

**得分**：88/100 🟢（代码修正后+15分）

---

### 一、名词解释（每题5分，共20分）
1. **字典和列表的区别**     //作答：字典最适合存放有关联的数据的，列表不是 🟡 补充：列表有序用索引访问，字典用键访问键值对
2. **`d[key]` 和 `d.get(key)` 的区别**     //作答：d[key]如果key不存在会引发KeyError，而get方法返回None ✅
3. **`.items()` 方法的作用**     //作答：将字典的键和值组装成一个二元组 ✅
4. **`d["score"]=90` 新增 vs 修改**     //作答：字典里没有score时是新增，有则是修改 ✅

### 二、代码阅读（每题10分，共20分）
```python
d = {"a": 1, "b": 2}
d["c"] = 3
print(d.get("d", 0))  # 作答：0 ✅
print(d)              # 作答：{'a':3,'b':2,'c':1} ❌ 正确：{'a':1,'b':2,'c':3}
```
```python
text = "a b a c b a"
words = text.split()
count = {}
for w in words:
    count[w] = count.get(w, 0) + 1
print(count)  # 作答：{'a':3,'b':2,'c':1} ✅
```

result[item] = result.get(item, 0) + 1     // ✅

### 四、手写代码（30分）— 已修正重新评分：30/30 ✅
**题目**：函数phone_book(name)返回电话号码，不存在返回"未找到此人"

🟢 **你修改后的代码（满分）：**
```python
def phone_book(contact_name):
    pbook = {'Alex': '11111', 'Deram': '25468', 'Tom': '4678964', 'Steve': '564982313'}
    if contact_name not in pbook:
        return "Not found!"
    return f'The phone number is : {pbook[contact_name]}'

if __name__ == "__main__":
    contact_name = input("Who do you want to see :")
    print(phone_book(contact_name))
```
✅ 参数名有意义 `contact_name`  ✅ 用参数而非全局变量  ✅ 用`return`  ✅ 外部`print`接收返回值

### 五、概念辨析（20分）
| 概念A | 概念B | 作答与点评 |
|-------|-------|-----------|
| 字典的键 | 字典的值 | //作答：键是一个元素，值是有关的元素 🟡 键是唯一标识符(不可变)，值是键对应的数据 |
| `d.update({"x":1})` | `d["x"]=1` | //作答：update是合并，d["x"]=1是添加 🟡 单键操作效果相同；update可一次更新多个键 |

---

### ❌ 错误总结

| 错误 | 正确 | 原因 |
|------|------|------|
| 函数忽略参数name，用全局index_name | 用参数 `name` 而非全局变量 | 函数参数形同虚设（但能运行不报错） |
| 用print替代return | `return f"电话：{pbook[name]}"` | D5老问题：print只显示不返回 |
| `d = {"a":3,...}` 看错值 | `{"a":1,"b":2,"c":3}` | d["c"]=3只改c，不改a |

### 📌 核心教训
> **函数里用参数，别绕过去用全局变量；print显示不是返回！**
