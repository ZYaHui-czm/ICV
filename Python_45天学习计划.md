## 🔑 Python 最重要的 20%

这 20% 的知识覆盖了日常开发中 80% 的场景，**优先精通这部分**：

| 优先级 | 核心知识 | 占比 | 原因 |
|--------|---------|------|------|
| ⭐1 | **数据结构与容器**（list/dict/set/tuple、切片、推导式） | 5% | 所有程序的数据载体 |
| ⭐2 | **控制流与函数**（条件、循环、函数定义、参数类型、作用域） | 5% | 逻辑骨架 |
| ⭐3 | **字符串处理**（格式化、split/join、strip、正则基础） | 3% | 数据处理入口 |
| ⭐4 | **文件 I/O**（open/with、读写模式、路径处理） | 2% | 与外界交互 |
| ⭐5 | **异常处理**（try/except/else/finally、raise） | 2% | 健壮性基石 |
| ⭐6 | **模块与包**（import、pip、虚拟环境、`__name__`） | 1% | 工程化基础 |
| ⭐7 | **函数式工具**（lambda、map/filter、列表推导式进阶） | 1% | 简洁高效 |
| ⭐8 | **面向对象基础**（类、继承、`__init__`、self、特殊方法） | 1% | 理解框架/库 |

> **结论**：前 3 项（数据结构、控制流、字符串）就占了约 60% 的日常使用，**先死磕这 3 项**。

## 📊 五级学习体系

### Lv.1 — 基础感知（第1-6天）

**目标**：能写 30-50 行的单文件脚本，处理简单的数据变换。

#### 必须掌握
- Python 环境安装与运行（IDLE / VS Code / 终端 `python` 命令）
- 变量与数据类型：`int`、`float`、`str`、`bool`、`None`
- 基本输入输出：`print()`、`input()`
- 运算符：算术、比较、逻辑 (`and`/`or`/`not`)
- 字符串操作：拼接、`f-string`、`len()`、索引、切片 `s[i:j]`
- 列表：创建、索引、切片、`append/pop/remove/sort`
- `for` 循环与 `range()`、`while` 循环
- `if/elif/else` 条件判断
- 定义函数：`def`、`return`、参数

#### 常见错误
| 错误 | 正确做法 |
|------|---------|
| `=` 和 `==` 混淆 | `=` 赋值，`==` 比较 |
| 索引从 1 开始 | Python 索引**从 0 开始** |
| 切片的 `[i:j]` 不包括 j | `s[0:3]` 取索引 0,1,2 |
| `list` 作为函数默认参数 | 用 `None` + 内部赋值替代 |
| 忘记冒号 `:` | `if x > 0:`、`for i in range(5):` |
| 混用空格和 Tab | **统一用 4 空格** |

#### ✅ 进入 Lv.2 的标准
- 能独立写出：猜数字游戏、简单计算器、列表统计脚本
- 无语法错误写出 `for`/`if`/`def`
- 理解切片 `s[::-1]` 是反转

---

### Lv.2 — 结构编程（第7-14天）

**目标**：能写 100-200 行的脚本，使用字典处理数据，处理文件读写，能捕获异常。

#### 必须掌握
- **字典**：键值对、`get/setdefault/update`、遍历 `.items()`
- **元组**：不可变性、拆包 `a, b = (1, 2)`
- **集合**：去重、交集/并集/差集
- 嵌套数据结构：列表套字典、字典套列表
- **文件 I/O**：`open()`、`with` 语句、`.read/.readlines/.write`
- **异常处理**：`try/except/else/finally`，捕获特定异常
- `enumerate()`、`zip()`、`in` 关键字
- 列表推导式 `[x**2 for x in nums]`
- 字符串方法：`.split/.join/.strip/.replace/.find`
- 模块导入：`import`、`from ... import`、`as` 别名
- pip 安装第三方库

#### 常见错误
| 错误 | 正确做法 |
|------|---------|
| 字典键不存在时崩溃 | 用 `.get(key, default)` 或 `in` 先判断 |
| 文件忘记关闭 | 始终用 `with open(...) as f:` |
| 捕获所有异常 `except:` | 捕获具体异常 `except ValueError:` |
| 混淆 `list.append` 和 `list.extend` | `append` 加一个元素，`extend` 合并列表 |
| 修改列表时遍历 | 遍历副本 `for x in lst[:]:` |

#### ✅ 进入 Lv.3 的标准
- 能读写 CSV/TXT 文件并处理数据
- 能用字典做词频统计
- 能写出带异常处理的输入验证

---

### Lv.3 — 工具化编程（第15-24天）

**目标**：能写 300-500 行的工具脚本，使用函数式编程范式和常用标准库，理解面向对象基础。

#### 必须掌握
- **函数进阶**：默认参数、`*args`/`**kwargs`、返回值类型、文档字符串
- **lambda 匿名函数**
- **`map()` / `filter()` / `sorted(key=...)`**
- **面向对象基础**：类定义、`__init__`、`self`、实例方法、类变量 vs 实例变量
- **特殊方法**：`__str__`、`__repr__`、`__len__`、`__eq__`
- **常用标准库**：
  - `os`：文件/路径操作
  - `sys`：命令行参数 `sys.argv`
  - `json`：`json.load/dump`
  - `datetime`：日期时间操作
  - `random`：随机数
  - `collections`：`Counter`、`defaultdict`
  - `math`：数学函数
- **虚拟环境**：`python -m venv venv`、激活、`pip freeze > requirements.txt`

#### 常见错误
| 错误 | 正确做法 |
|------|---------|
| 可变对象作为默认参数 | 用 `def f(lst=None):` + `if lst is None: lst = []` |
| 混淆类变量与实例变量 | 类变量所有实例共享，实例变量用 `self.xxx` |
| 忘记 `self` 参数 | 实例方法的第一个参数必须是 `self` |
| pip 全局安装污染环境 | 总是先创建虚拟环境 |
| `os.system()` 执行命令 | 优先用 `subprocess` 模块 |

#### ✅ 进入 Lv.4 的标准
- 能写一个带命令行参数的文件处理工具
- 能用类封装数据和行为
- 能配置虚拟环境并管理依赖

### Lv.4 — 工程化编程（第25-34天）

**目标**：能使用第三方框架搭建 Web 应用或数据管道，理解调试与测试。

#### 必须掌握
- **装饰器基础**：`@staticmethod`、`@classmethod`、简单自定义装饰器
- **迭代器与生成器**：`yield`、`next()`、生成器表达式
- **上下文管理器**：`with` 原理、自定义 `__enter__/__exit__`
- **pip 进阶**：`requirements.txt` 版本锁定、`pip install -e .`
- **常用第三方库**：
  - `requests`：HTTP 请求
  - `pandas` 基础：`DataFrame`、`read_csv`、基本筛选
  - `matplotlib` 基础：基本折线图、柱状图
  - `flask` 或 `fastapi` 基础：路由、请求/响应
- **调试技巧**：`print` 调试 → `pdb` → IDE 断点调试
- **单元测试基础**：`pytest`、`assert`、测试函数命名
- **Git 基础**：`init/add/commit/push/pull`、`.gitignore`

#### 常见错误
| 错误 | 正确做法 |
|------|---------|
| 不用虚拟环境开发项目 | 每个项目独立虚拟环境 |
| 测试只测"快乐路径" | 也要测试边界情况和异常输入 |
| 装饰器不理解执行顺序 | 装饰器从内到外执行 |
| 生成器耗尽后复用 | 生成器只能迭代一次 |
| 硬编码路径/密码 | 用环境变量 + `.env` 文件 |

#### ✅ 进入 Lv.5 的标准
- 能搭建一个简单的 Flask/FastAPI Web 应用
- 能用 pandas 读取和处理真实 CSV 数据
- 能写至少 5 个 pytest 测试用例

---

### Lv.5 — 独立项目（第35-45天）

**目标**：独立完成一个完整项目，从需求分析到部署。

#### 必须掌握
- **项目结构**：标准 Python 项目目录布局
  ```
  my_project/
  ├── README.md
  ├── requirements.txt
  ├── setup.py 或 pyproject.toml
  ├── my_project/
  │   ├── __init__.py
  │   ├── main.py
  │   └── utils.py
  └── tests/
      └── test_main.py
  ```
- **面向对象设计**：单一职责、封装、组合优于继承
- **日志系统**：`import logging`，配置级别/格式/输出
- **配置文件管理**：`config.py`、环境变量、`.env`
- **异常链与自定义异常**
- **API 调用与数据处理**（从公共 API 获取数据并处理）
- **简单的数据可视化**（用 matplotlib 或 seaborn 生成报告）
- **代码规范**：PEP 8、类型提示基础、`black`/`flake8`
- **部署基础**：`gunicorn` / uvicorn、简单 Dockerfile

#### 项目选题建议（选 1 个完成）
1. **命令行笔记管理工具** — 增删改查 + 标签 + 搜索 + JSON 持久化
2. **天气查询 CLI** — 调用公开 API，缓存结果，格式化输出
3. **个人记账小工具** — CSV 导入/导出，分类汇总，图表展示
4. **简易博客系统** — Flask + 模板 + SQLite 存储
5. **批量文件整理工具** — 按扩展名/日期分类，自动重命名

#### ✅ Lv.5 毕业标准
- 项目代码在 GitHub 上可访问
- 有 README 文档和 `requirements.txt`
- 包含至少 10 个测试用例且通过
- 能向他人解释项目架构和设计选择

## 🎯 考核与复盘说明

### 每日复盘流程
1. **我出题** — 每天学习结束后，我生成一份当日复盘考核文档
2. **你作答** — 你在文档中填空/写代码/回答问题
3. **我批改** — 我逐题评分，指出：
   - ✅ 完全正确的地方
   - ❌ 错误的地方及原因
   - 🔍 理解缺口（知道概念但说不清）
   - 🔄 混淆的内容（两个概念搞混了）
   - 📝 未能正确理解的高级词汇
4. **错题总结** — 我把所有错误汇总成本日错题集

### 每周复盘格式
每周末生成一页式复习速查表，内容包括：
- 主题一句话定义
- 核心概念（列表/要点）
- 代码样例（关键用法）
- 常见错误（本周最高频错误）

### 问题问答机制
当你问我任何 Python 问题时：
1. **我先用大白话解释**，不用高级术语
2. **你用自己的话复述一遍**
3. **我指出**：对错、遗漏、混淆、未能真正理解的词汇
